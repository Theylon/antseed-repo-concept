How It Works

Install. Route. Unstoppable.

AntSeed runs as a local proxy. Your tools send requests to localhost. The protocol handles everything else.

01

Install the daemon

One command. Runs as a background service. Exposes an API-compatible endpoint on localhost. Your existing tools work without modification.

02

Set your preferences

Tell AntSeed what you care about: cheapest price, lowest latency, specific capabilities, TEE privacy. The router scores all providers and picks the best match.

03

Never stop working

Hit a rate limit? Provider down? AntSeed detects it instantly and reroutes to the next best provider. Automatic failover across a decentralized network.

Supply Side

Anyone can provide.
No partnership required.

The protocol is provider-agnostic. It does not care how a seller fulfills a request. It cares that a response came back, the receipt verified, and quality was consistent.

Skilled Inference

Anyone with frontier model API access can offer specialized agents with Skills — legal research, security audit, market analysis. Compete on outcomes, not just price.

Self-Hosted Operators

A gamer with a GPU, a developer with a Mac Mini running open-weight models. No terms-of-service issues. Cost basis is electricity and hardware depreciation.

Inference Farms

Professional operators running optimized stacks at scale. Always on, high throughput, high stake. Farms in cheap-electricity regions set the global floor price.

Privacy / TEE Nodes

Trusted Execution Environments where not even the operator can see prompts. Cryptographic attestation proves the enclave is genuine.

Custom Model Operators

Models serving use cases that cannot exist on centralized platforms — security research, red-teaming, unrestricted creative work.

Edge Providers

Metro-located nodes optimized for speed. For real-time coding assistants, chatbots, and agentic chains where every millisecond matters.

Evolution

One protocol. Three markets.

Each builds on the one before it. All three share the same discovery, routing, reputation, and settlement mechanisms.

01

Commodity Inference

A seller has a model or API access. A buyer needs inference. They trade directly. Price set by open competition — margins compress toward zero and the buyer pays near-cost.

02

Differentiated Services

Same protocol, but the seller equips their model with Skills — modular packages of domain expertise and workflows. The buyer does not care what is inside. They care about the result.

03

Agent Commerce

Same protocol, but now the buyers are also machines. An autonomous agent holds credits, discovers providers by capability, evaluates reputation, consumes services, and settles payment.

Protocol Overview

Five Layers, Direct Settlement

Discovery, transport, metering, payments, and reputation happen peer-to-peer with no central marketplace operator in the request path.

L1

Discovery

DHT peer discovery and capabilities lookup.

L2

Transport

WebRTC/TCP binary framing and streaming.

L3

Metering

Token estimation plus signed receipts.

L4

Payments

USDC escrow channel settlement.

L5

Reputation

Local + attestations-based trust score.

Message Types

0x01 / 0x02Handshake init + ack
0x10 / 0x11Ping + pong keepalive
0x20Buyer HTTP request
0x21 / 0x22 / 0x23Response full / chunk / end
0x24Response error
0xF0 / 0xFFDisconnect / protocol error

Settlement Formula

Every request meters usage and settles via escrow receipts.

totalCostUSD = inputTokens * inputRate + outputTokens * outputRate
protocolFeeUSD = totalCostUSD * feeRate
sellerPayoutUSD = totalCostUSD - protocolFeeUSD

Channel lifecycle: open → active → disputed → settled → closed
